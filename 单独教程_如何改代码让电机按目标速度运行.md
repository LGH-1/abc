# 单独教程：如何改代码让电机按目标速度运行（带序号注释）

> 目标：你可以直接按这份教程改工程，让电机上电后自动启动并跑到目标速度。
> 适用：当前这套 ST MCSDK 风格工程（`main.txt` / `mc_tasks.txt` / `mc_config.txt` 提取版）。

---

## 0. 先理解：你要改哪几处

你只需要改 3 类地方：

1. **目标速度参数**（默认转速）
2. **main 里自动启动逻辑**（上电后自动发启动 + 速度斜坡命令）
3. **（可选）速度环 PI**（如果不稳再调）

---

## 1) 改默认目标速度（第一步）

在 `mc_config.c`（你的提取文件是 `mc_config.txt`）里，`SpeednTorqCtrlM1` 结构体有默认速度成员。

你要确认/调整：
- `MecSpeedRefUnitDefault`
- `MaxAppPositiveMecSpeedUnit`
- `MinAppPositiveMecSpeedUnit`

> 建议先把目标设在额定速度的 10%~20%，先求稳定。

---

## 2) 在 `main.c` 加“只执行一次”的自动启动代码（核心）

> 下面给的是**可直接参考的写法**，每个改动位置都带了“序号注释”。

### 2.1 在全局变量区添加启动标志

放在 `/* USER CODE BEGIN PV */` 内：

```c
/* USER CODE BEGIN PV */
static uint8_t auto_start_done = 0U;   // [修改1] 自动启动只执行一次的标志
/* USER CODE END PV */
```

### 2.2 在 `USER CODE BEGIN 2` 定义目标速度（可读性更好）

```c
/* USER CODE BEGIN 2 */
#define TARGET_SPEED_UNIT   (300)      // [修改2] 目标速度（单位是 MecSpeedUnit，不一定等于 rpm）
#define TARGET_RAMP_TIME_MS (1000)     // [修改3] 速度斜坡时间，避免瞬间冲击
/* USER CODE END 2 */
```

### 2.3 在 `while(1)` 里加入一次性启动命令

放在 `/* USER CODE BEGIN 3 */` 内：

```c
/* USER CODE BEGIN 3 */
if (auto_start_done == 0U)
{
  MCI_StartMotor(&Mci[M1]);                                   // [修改4] 触发电机启动状态机
  MCI_ExecSpeedRamp(&Mci[M1], TARGET_SPEED_UNIT, TARGET_RAMP_TIME_MS); // [修改5] 下发目标速度+斜坡
  auto_start_done = 1U;                                       // [修改6] 防止循环里反复发启动命令
}
/* USER CODE END 3 */
```

---

## 3) 如果“能转但不稳”，再改速度环 PI（第二阶段）

在 `mc_config.c`（提取文件 `mc_config.txt`）里找到：
- `PIDSpeedHandle_M1.hDefKpGain`
- `PIDSpeedHandle_M1.hDefKiGain`

调参顺序建议：
1. 先小 `Kp`、小 `Ki` 能稳定运行；
2. 再逐步加 `Kp` 提升响应；
3. 有静差再加一点 `Ki`；
4. 抖动就先降 `Kp`，再降 `Ki`。

---

## 4) 你最容易踩坑的地方（务必看）

1. `TARGET_SPEED_UNIT` 不是 rpm，单位来自工程速度单位换算（`SPEED_UNIT` 体系）。
2. 若霍尔相位/极对数设置不对，速度再怎么改也会抖或反转。
3. 不要在 `while(1)` 每次循环都发 `MCI_StartMotor()`，必须像上面那样只发一次。
4. 先别动电流环 PI（`PIDIq/PIDId`），先把速度环跑稳。

---

## 5) 最小改动清单（复制后逐项打勾）

- [ ] 改默认速度参数（`MecSpeedRefUnitDefault`）
- [ ] `main.c` 增加 `auto_start_done`（[修改1]）
- [ ] 加 `TARGET_SPEED_UNIT`、`TARGET_RAMP_TIME_MS`（[修改2][修改3]）
- [ ] 在主循环增加一次性启动代码（[修改4][修改5][修改6]）
- [ ] 低速验证稳定后，再提高目标速度

---

## 6) 对照当前工程调用链（帮助你确认改对了）

- `main()` 完成初始化；
- `MCboot()` 做控制对象初始化，并会初始化 PID / 传感器；
- `ADC1_2_IRQHandler()` 触发 `TSK_HighFrequencyTask()`；
- `FOC_CurrControllerM1()` 执行电流环核心；
- 速度目标由 `MCI_ExecSpeedRamp()` 下发后，状态机与速度环自动闭环跟踪。
